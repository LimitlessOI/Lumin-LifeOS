# Architect Comparison Rubric

Score each category `0-5`.

- `0` = missing or wrong
- `1` = mentioned but weak
- `3` = materially present
- `5` = precise, enforced, and better than current canonical output

## Categories

1. **Supreme-law alignment**
   Does the architect preserve North Star, Companion, vocabulary law, and founder packet law without creating a competing authority chain?

2. **Point-B truth**
   Does it explicitly prevent `TECHNICAL_PASS` from being treated as Point B completion?

3. **Queue determinism**
   Does it preserve `BP_PRIORITY.json` as the only active product queue and eliminate false-green idle states?

4. **Builder no-decision law**
   Does it prevent Builder/CDR from making product, architecture, UX, scope, or acceptance decisions?

5. **Self-repair escalation law**
   Does it encode three-attempt stages, lessons carry-forward, research, and added consensus/model escalation honestly?

6. **Receipt/twin sync**
   Does it force reality changes to update receipts, queue truth, and twin-state artifacts?

7. **UI proof**
   Does it require proof through the same founder UI surface Adam uses?

8. **Authority hygiene**
   Does it clean stale authority tags and keep Hist-owned artifacts out of active runtime authority?

9. **Parts-car intelligence**
   Does it preserve the good working pieces and retire the bad ones cleanly?

10. **Mechanical enforcement**
   Does it turn important doctrine into shared helpers, gates, tests, and scripts instead of prose only?

11. **Determinism honesty**
   Does it require same-tier determinism rather than stronger-model theater?

12. **Certification honesty**
   Does it keep `FULLY_MACHINE_READY` false until proof is real?

13. **Commit transport truth**
   Does it prevent build/repair PASS without exact commit-proof semantics?

14. **Deploy/runtime truth**
   Does it distinguish local/build success from live runtime success and require deploy-SHA or equivalent runtime proof?

15. **Founder-surface completeness**
   Does it close the gap where API/scripted probes pass while the exact founder prompt/flow still fails?

## Comparison Questions

1. What did the external architect catch that the canonical pack missed?
2. What did the canonical pack catch that the external architect missed?
3. Which pack left more room for coder interpretation?
4. Which pack had the better parts-car salvage judgment?
5. Which pack had the better self-repair law?
6. Which pack was more mechanically enforceable?
7. Which pack was more honest about current-state gaps?
8. Which pack handled commit/deploy proof more rigorously?
9. Which pack was less likely to miss a founder-only failure mode?

## Promotion Rule

Promote the stronger idea per category.
Do not keep a weaker canonical answer for ego.
If the external architect is better in a category, replace the weaker canonical answer and receipt the change.
