# C2 Module Charter

**Canonical anti-drift doc:** `docs/C2_CANONICAL_DEFINITION.md`

## Founder's definition

C2 is Adam's **bridge to communicate with the system** — eventually a module **inside LifeOS** (one convenient place), not a separate brain or department.

## Formal name

C2 = **Command, Control, and Communication** — a LifeOS module.

## C2 must

- expose communication with the system (send + receive)
- expose founder action queue (what needs Adam)
- expose mission state (what is running, blocked, done)
- expose critical escalations (narrow criteria only)
- allow density expansion and contraction (headline ↔ detail)

## C2 must not

- act as the strategy engine (that is Product Development + Council)
- act as the blueprint engine (that is BPB)
- act as the builder (that is Coder / BuilderOS)
- declare truth or "done" (that is SENTRY + receipts)
- assign Builder work
- replace the AI Council

## Category lock

- C2 is **not** a department.
- C2 is **not** a separate product outside LifeOS.
- C2 does **not** assign Builder work.

## Runtime note

Command Center (`AMENDMENT_12`) is the current partial implementation of C2 on Railway. Cursor is an interim bridge until LifeOS C2 is complete.
