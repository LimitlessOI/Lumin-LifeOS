Council Mandate and Detection Capabilities

This overlay displays the Kingsman council mandate and the current detection surface for submitted threat patterns. It is aligned to the existing Kingsman detect route and the Kingsman home SSOT.

Council Mandate

Authoritative intent
Mandate: coordinate council-backed review of threatening patterns and preserve a clear receipt trail for each submitted detection payload.
  • Owner-scoped intake: every submission is tied to the authenticated user sub.
  • Structured evidence: both pattern and evidence are required inputs.
  • Receipt-first flow: successful submissions return the stored record with timestamps.

Detection Capabilities

Available now

Pattern capture

Accepts a JSON payload describing the pattern being detected.

Evidence capture

Accepts a JSON payload for supporting evidence alongside the pattern.

Persistence

Stores records in kingsman_threat_patterns and returns the created row.

Submission Flow

POST /api/v1/detect
Step Behavior
Authenticate Requires key access and a user JWT subject.
Validate Rejects requests missing pattern or evidence.
Persist Inserts a new detection row and updates timestamps.
Return Responds with { ok: true, data } on success.

Detection Response Surface
