### INTENT_COVERAGE_MAP

Tracks whether intent is sufficiently clarified for ARC. Coverage does not mean "mentioned." Coverage means the dimension is clear enough that a low-context third party can explain it without asking Adam another question, and ARC can translate it without inventing intent.

A dimension is sufficient only when:

1. A third party can restate it correctly.
2. It can be used to guide a decision.
3. It reduces downstream invention.
4. It can be tested, challenged, or traced.
5. It supports Adam getting what he intended from the conversation and simulations.

Coverage levels: MISSING (not addressed), MENTIONED (touched but not usable), PARTIAL (usable but ambiguous), SUFFICIENT (clear enough for ARC), LOCKED (explicit founder/council decision), PARKED (known gap intentionally deferred with risk owner).

Required dimensions: outcome, user, pain, value, success metric, failure metric, behavior change, constraints, tradeoffs, alternatives, assumptions, evidence needed, risk, dependencies, stage target, done definition, rollback condition, ownership, scope boundary, priority fit, competing intents, market reality, future implications, Adam desired result, unacceptable result, alpha learning target, release boundary.

**Minimum sufficiency rule:** IDC may not exit if any load-bearing dimension is MISSING or only MENTIONED.

If a dimension is PARKED, the packet must state why it is safe to park, who owns it, when it must be resolved, and what downstream work is blocked by it.

## CONSENSUS DOCTRINE

Consensus is mandatory on load-bearing decisions. Consensus is not A-versus-B voting — it exists to create the best available decision, which may be A, B, a refinement, a combination, or a new synthesis discovered through debate.

Dissent is encouraged. Different paradigms are required. Disagreement is evidence generation.

The system seeks the fastest honest path to consensus that preserves challenge. Fast path means reaching consensus quickly when evidence and steel-man already resolve dissent — it does not mean skipping consensus.

Founder escalation is rare and only for unresolved intent ambiguity. Mechanical, engineering, vendor, architecture, cost, implementation, or optimization disagreements do not escalate to the founder. The council continues until consensus, halt, or recorded blocker.

Full council includes every department. Chair leads the council and participates as a full contributor. Chair decides when a decision is important enough for full-council involvement — reserved for load-bearing, cross-functional, constitutional, high-risk, or strategically important decisions.

Load-bearing means changing the decision later would likely affect: intent fidelity, more than one department, architecture, user experience, cost/timeline materially, public release, constitutional law, safety/trust, multiple downstream decisions.

## ANTI-DEADLOCK RULE

If consensus cannot be reached after serious challenge:

* If the dispute is mechanical and reversible, choose the lowest-risk reversible path and record dissent.
* If the dispute is mechanical and irreversible/load-bearing, HALT with recorded blocker and required evidence.
* If the dispute is unresolved founder intent, return `BLOCKED_INTENT_AMBIGUITY` to IDC.
* Do not bring Adam into mechanical disputes.
* Do not loop forever.

## ALPHA FEEDBACK PROTOCOL

After Builder execution and initial validation, Adam should not be pulled back into routine process until alpha testing unless unresolved intent ambiguity exists. Alpha is a major intent refinement stage.

Feedback is classified as:

* **A. Intent Delta** — the intended outcome changed or became clearer. Requires INTENT_DELTA. Return to IDC or ARC.
* **B. Mechanical Fix** — intent unchanged, implementation adjustment only.
* **C. Studio Fix** — intent unchanged, but experience/feel/aesthetic/interaction failed. Return to Studio under ARC.
* **D. Market/User Reality Finding** — external reality changed or invalidated an assumption. Return to IDC with Wisdom/SNT/CFO/Chair review.

SNT owns alpha feedback classification. Builder records raw feedback. Adam may escalate severity upward but should not be required for routine classification.

## BUILD_PASS VS RELEASE_PASS

**BUILD_PASS** means the system built according to the current Blueprint / Twin. Built is not sellable. Sellable requires release proof.

NORTH_STAR.md Article I §1.1 (Healing — "no charge for healing, ever"), §1.2 (Education — "equal in weight to every other constitutional commitment"), and Article V-B (Hardship Protocol — "cannot be removed") are constitutionally permanent. V2's CFO ROI logic, Priority Map, and Anti-Deadlock "lowest-risk reversible path" rule have no clause exempting these three from cost/ROI/priority scoring. This is the same gap found in Codex's `ADAM_DECISION_FILTER_V1.md` — both documents independently omit a Motivation Model override.
**Fix:** Add a "Constitutional Floor" clause: CFO and PRIORITY_MAP scoring never demotes or blocks Healing/Education/Hardship work on ROI grounds alone; these three may only be deprioritized on capacity grounds, never on value grounds, and any such deprioritization requires a named receipt.

### 3. PRIORITY_MAP has no subordination rule to BP_PRIORITY.json — **MEDIUM-HIGH** — KNOW
The repo already has a live canonical priority file (`builderos-reboot/BP_PRIORITY.json`, modified this session). V2 introduces `PRIORITY_MAP` as a CFO+Chair artifact without stating its relationship to that file — same "competing queue authority" failure mode Codex's own `CONTRADICTION_REPORT_ADF_V1.md` (#3) flags for BP queue vs QUICK_LAUNCH.
**Fix:** State PRIORITY_MAP is a read-only projection feeding into BP_PRIORITY.json, never a second source of truth; BP_PRIORITY.json remains canonical per existing AGENT_RULES.

### 4. IDC Mode A has no forcing function into Mode B — **MEDIUM** — THINK
Nothing in the packet specifies who decides Creative Expansion is "done" and Reality Translation must start. Without an explicit trigger, Mode A could run indefinitely (founder-favorable but progress-stalling) or get cut short by an eager CFO/SNT (intent-fidelity-risking).
**Fix:** Chair owns the Mode A→B transition call, logged as a one-line receipt with rationale; either Adam or SNT can force early transition if drift risk is identified.

### 5. Coverage levels are self-graded with no independent check — **HIGH** — KNOW
INTENT_COVERAGE_MAP's five levels (MISSING/MENTIONED/PARTIAL/SUFFICIENT/LOCKED/PARKED) are assigned by IDC itself with no second-party check before the IDC Exit Gate fires. This is structurally the same flaw as Codex's `PREDICTIVE_ACCURACY_REPORT_ADF_V1.md` — a 92.5% accuracy figure graded by the same model that wrote the rules being tested, no independent citations.
**Fix:** SNT Review #1 must independently re-grade at least the load-bearing dimensions before IDC Exit Gate can close, not just attack intent quality. Disagreement between IDC's self-grade and SNT's grade is itself a required receipt field.

### 6. Model Meritocracy has no minimum sample size or cool-down — **MEDIUM** — THINK
"No model permanently owns any role... earn, retain, lose, or regain roles through measured performance" has no stated minimum number of scored decisions before a role can change hands, risking role-thrashing on a single bad/good outcome.
**Fix:** Require Hist to accumulate at least N (suggest 5) scored decisions in a role before a reassignment recommendation is actionable, and a cool-down period after any reassignment.

### 7. "Predicted Adam never overrides Actual Adam" is undefined when Adam is unreachable — **HIGH for unattended ops** — KNOW
CLAUDE.md's own standing order is push-by-default/autonomous execution. V2's Adam Filter doctrine is correct for when Adam is present, but never states what happens when a decision needs founder-level judgment and Adam is asleep, traveling, or otherwise unreachable during an autonomous run.
**Fix:** Add explicit fallback: if a decision requires Adam-level judgment and Adam is unreachable, default to `BLOCKED_INTENT_AMBIGUITY` (hold, do not guess) rather than letting Predicted Adam stand in as a silent proxy — consistent with Article III's existing "fail-closed" instinct.

### 8. Builder Interchangeability has no test — **LOW** — KNOW
The doctrine ("a Blueprint must be strong enough multiple builders produce materially similar results") is asserted but not wired to any verification mechanism.
**Fix:** This already exists — `builderos-reboot/DETERMINISM_TEST_RUNBOOK.md` is live infrastructure in the repo. Cite it directly as the enforcement mechanism for this doctrine rather than leaving it as an unattached assertion.

### 9. Hardship-detection trigger is inherited silently, not named as an open gap — **MEDIUM** — THINK
Article V-B requires the system to detect financial hardship and stop charging automatically, but V2 never names who/what owns the detection trigger (CFO? Wisdom? a billing service?) even though it inherits the obligation by reference.
**Fix:** Name CFO as owner of the hardship-detection signal definition, with SNT auditing that the trigger actually fires before any product surface goes to a paid tier.

### 10. Memory section states a provisional answer as settled while also asking it as an open Reviewer Question — **LOW** — KNOW
The Memory section asserts "Provisional V2 decision: Memory is a platform capability under Wisdom" as if resolved, then Reviewer Question #13 asks the same thing as still open. Minor internal inconsistency, but it muddies what's actually settled vs. still under review.
**Fix:** Either drop Q13 or relabel the Memory section's provisional answer explicitly as "default-if-no-objection," not "decision."

### Verdict

None of these block the proposed **Chair + IDC + ARC Intake Loop v1** first lap, provided it stays supervised and away from money/billing/hardship/priority-reordering surfaces.

Items **1, 2, 3, 5, 7** are **BLOCKING** for any unattended run or any run that touches billing, priority reordering, or hardship-adjacent surfaces — these are the items where a wrong autonomous call has real founder-facing cost. Items 4, 6, 8, 9, 10 are non-blocking refinements.
