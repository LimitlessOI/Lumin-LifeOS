**Ethics tension (capture, don’t smuggle into build):** Brainstorm threads propose **Phantom shell**, **decoy surfaces**, **stealth** UX, and similar. **North Star** anchors **user sovereignty** and **non-manipulation**. Anything that sounds like **deceptive UI** needs **explicit legal / product ethics review** and **consent** framing — not silent implementation as “security.”

**Infra posture:** Historical **Tier 0 / Ollama / local-first council** ideas (**Stream Q**, parts of **J**) conflict with operator **cloud-first** posture (**Amendment 38** — GitHub / Railway / Neon). Catalog **both**; **default execution** follows **38** unless a **gated** exception is ratified.

**Competitive language vs system behavior:** **Q**-style dumps use **memetic warfare**, **velocity debt**, **Trojan-horse moats**, etc. That is **market/strategy framing** — it **does not** override **§2.6**-style truth, receipts, or misleading status in the **product**.

**`memory_crystals` (Grok):** Conceptual sketch aligning memory + council + cost — **not** a migrated DB schema without **gate-change** and **02** ownership.

**Revenue spine (often repeated):** Sequencing in dumps aligns with **ClientCare / billing recovery → TC → API cost / automation → BoldTrail** style ordering (**18 → 17 → 10 → 11**); verify live **INDEX** / **21** before treating any table as current.

**Evidence vs governance (Amendment 39):** The **evidence ladder** (CLAIM → … → INVARIANT) is **not** the **governance ladder** (Article → Amendment → rule). **INVARIANT ≠ LAW.** Brainstorm “constitutional” metaphors in **Q** must not blur that distinction.

**Operator-only context:** e.g. **Pewds**, hospital/family stress — stays under **21 → Operator personal context**; **never** mixed into impersonal backlog rows.

**Dump curls vs live system:** **Streams J / K / L** contain **historical** `curl` packs and SQL snippets. **`ENV_REGISTRY`**, **`SYSTEM_CAPABILITIES`**, and live **`routes/`** **win** for automation — dumps are **ideas + archaeology**.
