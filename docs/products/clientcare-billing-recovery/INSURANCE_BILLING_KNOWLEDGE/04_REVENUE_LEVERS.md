# 04 — Revenue levers (increase recovery legally)

**Definition of “increase the bill” here:** maximize **legitimate** dollars for care already given and contracted rates — not fabricate units.

---

## Lever 1 — Capture all billable work (biggest midwifery leak)

| Lever | What to do | Evidence needed | ClientCare |
|-------|------------|-----------------|------------|
| Don’t omit delivery global / delivery-only | Always create claim for birth | Birth note + DOS | ChargeSlip / SuperBill / HCFA |
| Don’t omit antepartum if itemized | Bill visit packages or E/M per rules | Visit notes | ChargeSlip |
| Don’t omit postpartum | Mother + baby as allowed | PP notes | Separate lines/claims |
| Bill newborn correctly | Separate patient/claim when required | Newborn chart | VERIFY |
| Problem visits outside global | E/M when medically necessary | SOAP | VERIFY global |
| Transfers / hospital | Bill what midwife still performed | Transfer note | May reduce delivery code — VERIFY |
| Supplies/meds if allowed | Only if unbundled by payer | Invoices | Often not |

**Future help tie-in:** birth/appointment mic → charting so this capture is automatic.

---

## Lever 2 — Code accuracy (right code &gt; bigger wrong code)

| Lever | Effect |
|-------|--------|
| Correct maternity code for what was done | Gets paid; wrong code denies |
| Correct DX supporting medical necessity | Prevents deny |
| Correct POS | Home vs office rates |
| Correct units / dates | Avoid return-to-provider |
| NCCI / bundling respect | Avoid takebacks |

Illegal upcoding or unbundling is **out of scope** and forbidden.

---

## Lever 3 — Eligibility & auth (prevent $0)

| Lever | Effect |
|-------|--------|
| VOB before birth | Know deductible, maternity coverage, in-network, auth |
| Fix member ID / DOB / subscriber | Cuts admin denials |
| Auth on file when required | Cuts “no auth” denials |
| Timely filing calendar | Late = $0 forever |

---

## Lever 4 — Clean claim submit (get it in the door)

| Lever | Effect |
|-------|--------|
| EDI via Ally with correct clearinghouse | KNOW Denise path |
| Insured vs patient fields correct | Denise: insured can be spouse |
| Taxonomy / NPI / pay-to | VERIFY practice setup |
| Secondary claims after primary | Captures second payer $ |

---

## Lever 5 — Post & reconcile every ERA (find missing $)

| Lever | Effect |
|-------|--------|
| Post remits completely | Know true balance |
| Compare paid vs allowed | Detect underpay |
| Patient PR billed promptly | Collect deductible |
| Secondary queued | Don’t leave $ on table |

## Ranking for Sherry’s practice (THINK)

1. **File every unbilled birth** (capture)  
2. **VOB + clean EDI** (prevent deny)  
3. **ERA reconcile + underpay chase** (recover)  
4. **Secondary + patient PR** (remainder)  
5. **Outside-global add-ons** (labs / NST / complication E/M / newborn / problem lactation) — see `08_GLOBAL_VS_ITEMIZED_RESEARCH.md`  
6. **Do not expect itemizing `59426+59409+59430` to beat ~$4,900 global** for the same full episode  
7. **Contract rates** (structural)
